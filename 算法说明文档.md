# DCF 估值模型 — 算法说明

> 基于蒙特卡洛模拟的两阶段 FCFF 贴现模型

---

## 一、核心思路

传统 DCF 的问题：给一组固定假设 → 算一条路径 → 得出一个"精确"的数字。但所有假设都是猜的，精确是假的。

本模型的做法：**不猜一个值，而是猜一个分布**。

```
不是：增长率 = 10% → FCFF = 412亿 × 1.1^t
而是：增长率 ~ 三角分布(min=2%, mode=10%, max=20%)
      → 每次模拟从分布中随机抽一个值
      → 5000 次模拟 = 5000 条可能的 FCFF 路径
      → 5000 个股价构成分布
      → 均值 = 目标股价
```

---

## 二、FCFF 怎么算的？

### 2.1 核心公式

不拿"FCFF 历史值"直接外推。每一轮模拟都是从**收入增长**开始，逐项重建 FCFF：

```
第 t 年：
  Revenue_t     = 基期收入 × (1 + 增长率)^t
  EBIT_t        = Revenue_t × 营业利润率
  FCFF_t        = EBIT_t × (1 - 税率) + D&A_t - 资本支出_t - 营运资本增加_t
```

每一项都是收入乘以一个比例，每个比例都是随机变量：

| 组件 | 占收入比例 | 随机分布 |
|------|-----------|---------|
| 营业利润 (EBIT) | 营业利润率 | 正态分布(历史利润率, 2%) |
| 折旧摊销 (D&A) | D&A/收入比 | 三角分布(2%, 5%, 10%) |
| 资本支出 (CapEx) | 资本支出/收入比 | 三角分布(2%, 6%, 15%) |
| 营运资本增加 (ΔWC) | 营运资本/收入比 | 三角分布(0%, 3%, 8%) |

## 三、从 FCFF 到股价（每轮模拟）

每一轮模拟的完整计算链：

```
                        ┌─ 增长率 ─┐
  基期收入 ───────────→ Revenue_t ─→ EBIT_t ─┐
                        ├─ D&A%   ─→ D&A_t   ├→ FCFF_t ─→ TV
                        ├─ CapEx% ─→ CapEx_t ┘          └→ PV(FCFF) + PV(TV) = EV
                        └─ WC%    ─→ ΔWC_t                     ↓
                                                    EV - 债务 + 现金 = 股权价值
                                                             ↓
                                                      股权价值 / 总股本 = 每股价格
```

具体计算：

**1. 终值**（假设企业永续经营）：
$$TV = \frac{FCFF_5 \times (1 + g_{perp})}{WACC - g_{perp}}$$

**2. 折现**：
$$PV(FCFF_t) = \frac{FCFF_t}{(1+WACC)^t} \quad PV(TV) = \frac{TV}{(1+WACC)^5}$$

**3. 企业价值 → 每股价格**：
$$EV = \sum PV(FCFF_t) + PV(TV)$$
$$Equity = EV - 有息负债 + 现金$$
$$Price = Equity / 总股本$$

## 四、5000 次模拟之后

```
5000 次重复 → 5000 个股价 → 分布统计：
                            │
              ┌──────────────┼──────────────┐
              ↓              ↓              ↓
         均值=目标价    P5/P95=90%区间  上涨概率
```

| 输出 | 含义 |
|------|------|
| 均值 197.90 元 | **目标股价**，所有模拟的平均 |
| 中位数 194.81 元 | 分布中心，接近均值→分布对称 |
| P5 149.29 / P95 255.44 | 90% 概率股价落在此区间 |
| 标准差 32.60 | 不确定性宽度 |
| 上涨概率 100% | 模拟中股价超过当前价的比例 |

## 五、支持性分析

蒙特卡洛产生目标价之后，再做两层交叉验证：

**敏感性分析**：固定 WACC 和永续增长率，算一个二维价格矩阵。回答"如果 WACC 变成 12% 会怎样"。

**情景分析**：悲观/基准/乐观三组假设各跑一次 DCF（不随机）。回答"如果整体变差会怎样"。

这两种分析用的**是同一套逐项拆解公式**，只是参数固定为场景值而非随机采样。

---

## 六、为什么这样设计

| 问题 | 解决方法 |
|------|---------|
| 增长率猜不准 | 三角分布(min=2%, mode=10%, max=20%) |
| 利润率会波动 | 正态分布(历史值, 标准差2%) |
| 某一年特殊值干扰 | 5000 次平均，异常值被稀释 |
| 模型过于敏感 | 分布宽度直观展示敏感度 |
| 报告只有单个数字 | 报告展示分布（P5~P95）和概率 |

---

## 七、局限

1. **各因子独立采样**：增长率和利润率不相关。现实中高增长常伴随低利润率（跑马圈地），这会略微增宽分布
2. **分布参数主观**：min/max 的范围是分析师判断
3. **终值占比高**：终值（第 5 年之后的价值）通常占 EV 的 60-80%，永续增长率假设影响大
4. **Beta 静态**：未做 Hamada 调整，默认 1.0
